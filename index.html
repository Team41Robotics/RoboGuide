Java Basics

On team 41 we use Java, an object-oriented programming language. What this means is that our programs are based around the creation and use of objects. These objects have properties which are associated with them. This allows us easily reference our data. For example, take a look at this simple Bird class: In Java, a class is an object that can be manipulated. In this case, the Bird object has attrributes that store its color, name, and position. It also has built in functions that manipulate its properties or return values about it. One advantage to creating an object such as this is that other users can use it without knowing exactly how it works. This principle is called abstraction and is very useful when writing programs. It allows people to easily build off pre-existing libraries (collections of programs).

Variables

One of the most important parts of any programming language is the use of variables. These are named values stored in the computer's RAM. In Java, the primitive variable types include the following:
  • boolean true or false (1 bit)
  • byte Stores whole number values from -128 to 127 (8-bit)
  • short Stores whole number values from −32,768 to 32,767 (16-bit)
  • char Stores character values from '\u0000' (0) to '\uffff' (65535) (16-bit). This is a way to store a single ASCII character.
  • int Stores whole number values from -2,147,483,648 to 2,147,483,647 (32-bit). If one tries to store a number with a decimal point in an int (or any other data type that cannot store floating point values), the compiler will throw an error.
  • long Stores whole number values from -9,223,372,036,854,775,808‬ to 9,223,372,036,854,775,807 (64-bit).
  • float Stores 32-bit floating point values (can have a decimal point)
  • double Stores 64-bit floating point values (can have a decimal point)
Another very commonly used data type in Java is a String. This type of variable is used to represent a String of character connected together. We use it to represent any collection of characters, including single characters, words, or sentences.
Also, it is a Java convention to declare variables using camel case.

Functions

The use of functions allows a programmer to organize their code effectively as well as reduce repition. Functions are sometimes also referred to as methods. Just like in mathematics, a function in Java has inputs (paramaters) and returns a value based on those inputs. In math you've probably seen a function written like this: This same function could be written in Java as: A function in java has a few main components that make it up:
  • Return Type - This includes all the primitive types talked about above, as well as any type of Object, or void (does not return anything)
  • Name - Typically written in camel case, like variables
  • Parameters - These are the inputs for the function. Each one must have its type specified. Functions can also be constructed without any parameters
  • Curly Braces - Not optional
  • Code to Execute - If the function has a return type (not void), then it MUST have a return statement in it.

Scope

Functions must placed inside a class. There are two main scopes that functions are declared with: public and private. Public functions can be accessed by any instance of the function's respective class object. Private functions can only be accessed within the specific class. This is useful for when a function is only used for some internal computation in the class. This will be shown further in the next section about classes.


Classes

Classes are essential components of programs in object oriented languages. An object in Java is created as a class. As mentioned in the beginning of the guide, a class can have multiple variables and methods associated with it.

FRC Programming Basics

File Structure

So now you've created a new Java project using the WPILib Timed Robot template. Where do you go to write your code? During this guide I am going to reference the final code for the 2020 robot a lot. You should open that up in another tab to follow along. In the 02_FinalRobot folder, you can navigate to src/main/java/frc/robot. In that folder you'll see a bunch of *.java files, two of which your template project already has: Main.java and Robot.java. The Main.java file is just there to start your program off (it's where you're main method is). You don't have to modify this at all. Where you can start to write some code is the Robot.java file. Also, when you create any additional classes they should be placed in the same parent folder as Main.java and Robot.java. The only other folder that you need to know something about is the vendordeps folder. It contains json files which allow you to import dependencies that are not part of WPILib (needed for SparkMax and TalonSRX/FX motor controllers).

Basic Robot Code

In Robot.java you'll find multiple methods including two that we use very often: robotInit() and teleopPeriodic(). We use robotInit() to initialize all of our variables when the program first runs. To control the robot, we use teleopPeriodic(). This method runs periodically (every 20ms) and executes whatever is inside of it. Because of that, loops are (usually) not required to make the robot move or sense the environment repeatedly. I'm sure you've been wondering how to actually make a robot move. It is now time to do that! To do so, we are going to use two different classes: Joystick and CANSparkMax. To initialize your Joystick object, find the port number of your controller in the Driver Station. The USB devices tab should look something like this: Your Joystick object should be created at the top of your program inside the Robot class but outside of any method. The method you will use to get data from your Joystick is JoystickName.getRawAxis(Axis #). You can find the axis numbers in the same page of the Driver Station. You will then feed the readings from your Joystick axes into your spark max's. The method you will use to set the speed is ... shocker ... SparkName.set(speed). If you feed two Joystick axes into at least two different sparks you should be able to make a robot move anywhere you want with tank drive! Motor Controllers

Talon SRX

  • The Talon SRX is a motorcontroller for use with brushed motors
  • Documentation: http://www.ctr-electronics.com/downloads/api/java/html/classcom_1_1ctre_1_1phoenix_1_1motorcontrol_1_1can_1_1_talon_s_r_x.html
  • Specs: https://www.ctr-electronics.com/talon-srx.html#product_tabs_tech_specs
  • Firmware: https://www.ctr-electronics.com/talon-srx.html#product_tabs_technical_resources
  • Setup: Phoenix Tuner
    • Wireless: Yes

  • Example from code: The shooterIntakeTalon in the Intake class from 2020
  • How to instantiate: TalonSRX exampleTalon = new TalonSRX(PORT)
  • How to import: import com.ctre.phoenix.motorcontrol.can.TalonSRX
  • Example uses:
    • Go forward (half speed): exampleTalon.set(ControlMode.PercentOutput, 0.5)
    • Stop: exampleTalon.set(ControlMode.PercentOutput, 0)
    • Go backward (full speed): exampleTalon.set(ControlMode.PercentOutput, -1)
  • Other Notes:
    • You also have to import the ControlMode class: import com.ctre.phoenix.motorcontrol.ControlMode
